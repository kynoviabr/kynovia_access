# Operational Pilot

This folder prepares Kynovia Access for a controlled operational pilot.

The pilot is a validation phase, not a production launch. It uses DEV data, mock providers, manual
observation, and documented acceptance criteria to confirm that the core access-control workflows are
ready for real-world feedback.

## Scope

- Define the pilot condominium profile.
- Prepare a deployment and readiness checklist.
- Provide realistic DEV seed data.
- Document operational test scripts.
- Define acceptance criteria.
- Collect gatehouse and resident feedback.
- Produce a post-pilot report and roadmap decision.

## Non-Goals

- No real hardware activation.
- No production rollout.
- No sensitive personal data.
- No new provider integrations.
- No automated gate opening from AI, LPR, facial recognition, or pilot scripts.
